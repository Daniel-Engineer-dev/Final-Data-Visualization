\chapter{Phương pháp phân tích và thiết kế}

\section{Quy trình phân tích}
Trình bày quy trình từ câu hỏi, dữ liệu, biến đổi, phân tích, trực quan hóa đến kiểm
chứng kết luận. Có thể thêm sơ đồ kiến trúc hoặc luồng dữ liệu.

\section{Phương pháp phân tích}
Mô tả các thống kê, phép tổng hợp, phương pháp phát hiện xu hướng, quan hệ giữa biến
hoặc mô hình được sử dụng. Nêu giả định và lý do lựa chọn.

\section{Thiết kế trực quan}
Với mỗi biểu đồ, giải thích:
\begin{itemize}
    \item Câu hỏi mà biểu đồ trả lời.
    \item Lý do chọn loại biểu đồ và visual encoding.
    \item Cách dùng màu sắc, nhãn, đơn vị và chú thích.
    \item Cách tránh gây hiểu nhầm hoặc quá tải thông tin.
\end{itemize}

\section{Storytelling và cấu trúc thông tin}
Mô tả thứ tự khám phá từ tổng quan đến chi tiết, các insight cần nhấn mạnh và cách
dashboard dẫn dắt người dùng.

\section{Kiến trúc hệ thống}
Trình bày frontend, backend, lớp dữ liệu, module AI, môi trường thực thi local và
nhật ký. Nêu rõ ranh giới giữa API AI và API thực thi.
